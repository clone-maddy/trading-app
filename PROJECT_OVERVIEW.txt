TRADING ASSISTANT: PROJECT OVERVIEW & ARCHITECTURE

This document provides a comprehensive overview of the Trading Assistant project. It is designed to be shared with other AI models (e.g., Claude) or developers to discuss future features, tools, and code improvements.

========================================================================
1. ARCHITECTURE & TECH STACK
========================================================================

The application is structured as a monorepo with separate frontend and backend services.

Tech Stack:
* Frontend:
  - React.js (SPA, Create React App)
  - State Management: Local React Hooks (useState, useEffect, useRef)
  - Real-Time Communication: socket.io-client (websockets)
  - Charts: recharts (custom implementation of Area/Line charts with SMA/RSI overlays and EMAReferenceDots)
  - Styling: CSS variables with a dark-mode theme
* Backend:
  - Node.js & Express.js
  - Database: MongoDB (Mongoose ODM)
  - Real-Time Server: Socket.IO (with secure user-specific rooms mapping JWT userIds to sockets)
  - Broker Integration: smartapi-javascript (official SDK for AngelOne SmartAPI)
  - Security & Auth: JSON Web Tokens (JWT), Speakeasy (TOTP validation)
  - Mailing & Onboarding: nodemailer (SMTP client)

========================================================================
2. CORE BACKEND SERVICES (/backend/services)
========================================================================

The backend coordinates live connections with the AngelOne broker and handles real-time positions tracking, risk management, and indicator calculations.

A. Broker Connection Module (backend/services/angelone.js)
* Purpose: Handles connection to AngelOne rest API endpoints.
* Key Operations:
  - connectAngelOne(): Authenticates via client credentials, password/MPIN, and generates a dynamic TOTP using the Speakeasy library. Caches JWT and Feed tokens.
  - getPositions(): Fetches live open/closed positions from the AngelOne account.
  - placeOrder(): Places market orders for real execution.
  - getTradeBook(): Fetches daily trade execution history.

B. WebSocket Feed Module (backend/services/websocketFeed.js)
* Purpose: Interfaces with AngelOne WebSocket Feed to stream live tick data and aggregate candles.
* Key Operations:
  - Connects using smartapi-javascript's WebSocketV2 client.
  - Aggregates token subscription requests across all UI clients (reference counting to avoid redundant broker queries) and background alert configurations.
  - Caches live LTP (Last Traded Price), Volume, Open Interest (OI), and Bid-Ask spreads.
  - Streams live updates to all connected frontend clients via a Socket.IO room.
  - Maintains a moving 1-minute candle cache, primes them from database subscriptions, aggregates live ticks, and triggers the alertsEngine crossover check on every tick and on candle close.

C. Option Chain Module (backend/services/optionChain.js)
* Purpose: Manages index and option contracts data.
* Key Operations:
  - Loads local scrip master metadata (scripmaster.json) to find NIFTY, BANKNIFTY, and FINNIFTY options contracts.
  - Queries expiries, extracts option chain layouts (Calls vs Puts per strike).
  - Fetches index spot prices and options candle data (1m, 5m, 15m, 1h, 1d timeframes) from AngelOne's historical APIs.
  - getSymbolByToken(token): Returns the formatted option contract symbol name for any token lookup.
  - getMultiExpiryOiSummary(): Performs concurrent 7-strike open interest summaries and Put-Call Ratio (PCR) calculations across the first 5 near-term expiries.

D. Real-Time Risk Monitor (backend/services/pnlMonitor.js)
* Purpose: Implements programmatic risk-management rules on open positions.
* Key Operations:
  - Runs a 1-second interval loop evaluating either Real or Virtual positions.
  - Global Settings: Auto-exits all positions when a combined target profit or stop-loss limit is reached.
  - Trailing Stop Loss: Automatically trails up the stop loss based on the peak P&L reached.
  - Partial Exits: Triggers partial position squared-offs (e.g., exit 50% quantity) at specified profit points.
  - Position-Specific Settings: Individual target profit, stop loss, and trailing parameters can be set per contract.
  - Retry Logic: Implements robust retry loops when placing square-off market orders to avoid execution failure.

E. Crossover Alerts Engine (backend/services/alertsEngine.js)
* Purpose: Evaluates server-side crossovers for configured stocks.
* Key Operations:
  - Evaluates EMA 9 and EMA 21 values on the candle series.
  - Detects crossovers: Bullish (EMA9 crosses above EMA21) and Bearish (EMA9 crosses below EMA21).
  - Includes a 60-second lockout timer to prevent notification spam from intra-candle price fluctuations (whipsaw protection).

F. Notification Dispatcher (backend/services/notificationDispatcher.js)
* Purpose: Delivers alerts to users across platforms.
* Key Operations:
  - Logs alert history records in MongoDB.
  - Pushes real-time Socket.IO alerts to active users.
  - Dispatches rich notification trading cards to the user's phone/desktop via Telegram Bot API when a telegramChatId is linked.

G. Telegram Polling Bot Service (backend/services/telegramBot.js)
* Purpose: Background polling service that queries Telegram bot updates for deep links.
* Key Operations:
  - Auto-clears webhooks on startup to prevent 409 API conflicts.
  - Evaluates incoming message starts `/start <userId>`, locates the MongoDB user, automatically links their chat ID, and sends a welcome confirmation.

H. SMTP Email Service (backend/services/emailService.js)
* Purpose: Integrates nodemailer SMTP connections with secure fallback mock logs.
* Key Operations:
  - Dispatches onboarding 6-digit verification code emails.
  - Sends a responsive welcome onboarding guide email upon successful account verification.
  - Supports platform-wide announcement/newsletter broadcasts to all users.

I. Indicator Utilities (backend/utils/indicators.js)
* Purpose: Mathematical definitions of technical indicators (calculateEMA).

========================================================================
3. DATABASE SCHEMA (/backend/models)
========================================================================

The database stores user profiles, authentication state, and logs for Virtual (Demo) Account performance and alerts:

1. User Schema (backend/models/User.js): Holds user name, email, password hashes, telegramChatId, tradingMode, `isVerified` flag, and `verificationCode`.
2. VirtualPortfolio Schema (backend/models/VirtualPortfolio.js): Holds demo account balances, win/loss stats, total P&L, and initial capital.
3. VirtualTrade Schema (backend/models/VirtualTrade.js): Logs individual paper trades, execution entries/exits, P&L, status (OPEN/CLOSED), and reasons for exit (e.g., TARGET_HIT, STOPLOSS_HIT, MANUAL).
4. Alert Schema (backend/models/Alert.js): Logs triggered alert history records (contract symbol, token, direction, EMA values, timestamp, seen flag).
5. AlertConfig Schema (backend/models/AlertConfig.js): Stores active alert monitoring subscriptions per user to run background websocket feed calculations.

========================================================================
4. FRONTEND VIEW & ROUTING SYSTEM (/frontend/src)
========================================================================

The client side is styled to provide a premium, real-time command dashboard split into three main environments:

Real Account Area:
* Real Dashboard (frontend/src/pages/RealDashboard.jsx): Lists live positions from AngelOne, tracks total P&L, displays global risk monitors (Target, SL, Trailing SL), and allows customizing individual stop-losses per contract.
* Real Option Chain (frontend/src/pages/RealTrading.jsx): Provides NIFTY/BANKNIFTY options contract tables with strike PCR summaries and a comparative multi-expiry 7-strike Open Interest comparison table.
* Real Analytics (frontend/src/pages/Analytics.jsx): Displays reports and charts of executed real trades.

Demo / Paper Trading Area:
* Virtual Dashboard (frontend/src/pages/VirtualDashboard.jsx): Simulates the real dashboard experience. Includes current virtual balance, win rate, total trades log, and options to reset the portfolio.
* Virtual Option Chain (frontend/src/pages/VirtualTrading.jsx): Allows simulated option buying or selling at current live market prices, with strike PCR summaries and a comparative multi-expiry 7-strike Open Interest comparison table.
* Virtual Analytics: Performance metrics for simulated strategies.

Alerts, Verification & Settings Center:
* Email Verification (frontend/src/pages/VerifyEmail.jsx): Security gate checking verification status on login. Prompts users for their 6-digit code and hides standard dashboard navigation menus if unverified.
* Alerts Dashboard (frontend/src/pages/Alerts.jsx): Displays triggered alerts log (mark-seen, delete actions), list of active stock configurations, and Telegram integration setup.
* Account Dashboard (frontend/src/pages/Account.jsx): Manage personal profile, MPIN, TOTP secrets, Telegram Chat IDs, and preferred default workspace. Includes one-click bot linkage and auto-detect status checking.
* Navbar unseen bell: A bell icon with a real-time badge counting unseen notifications, playing a chime on alert delivery.
* Navbar settings link: A settings gear button (⚙️) to open the Account page.

Live Chart Component (frontend/src/components/StockChart.jsx):
* Provides modal popups displaying selected contract candles.
* Enables timeframe changes (1m, 5m, 15m, 1h, 1d).
* Computes and overlays technical indicators on the client side:
  - Simple Moving Averages (SMA 9 and SMA 21).
  - Exponential Moving Averages (EMA 9 and EMA 21) line overlays.
  - Relative Strength Index (RSI 14) in a sub-chart.
* Supports setting custom price alerts.
* Overlays crossover ReferenceDots (▲ / ▼) directly on the chart lines at the exact timestamps where historical alerts triggered.

Here are concrete areas and tools to discuss for your next phase of development:

Semi-Automated / Algorithmic Trading Rules:
* Auto-Entry Strategies: Combine indicators (e.g., EMA crossovers, RSI breakouts) directly into order executors. For example: "If Nifty crosses below the 9 EMA, auto-buy the nearest OTM Put option".
* Multi-Leg Strategy Builder: Build support for multi-leg strategies (Straddles, Strangles, Iron Condors) with group-level target profit/stop-loss.

Notification Integrations:
* Webhooks: Push alerts to Discord or WhatsApp when a risk threshold is hit or positions are closed out.

Advanced Performance Analytics:
* Metrics: Calculate Sharpe Ratio, Drawdown, Profit Factor, and average hold times.
* Equity Curve: Visualize portfolio balance growth over time on a chart.

Developer Tooling & Testing:
* Historical Data Ingestion: Store historical option chain data in a MongoDB time-series collection to backtest strategies offline.
* Mock AngelOne Server: Create a local mock broker API server to simulate order failures, delay, and extreme slippage to test the resilience of your pnlMonitor exit logic.
